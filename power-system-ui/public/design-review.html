종합 점수 및 Executive Summary

🎯 전체 요약

PowerFlow Analyzer는 기술적 완성도 면에서 경쟁 제품 대비 80% 수준의 기능을 갖추었으나, 시각적 디자인과 UX는 프로토타입 시연 가능 수준이지 상용 출시 준비는 미완입니다. 핵심 엔지니어링 기능(Load Flow, Short Circuit, Cable Sizing, Harmonics 등)은 실무 엔지니어가 즉시 사용 가능하나, UI 레이어에서 첫인상을 망치는 3대 치명적 문제가 존재합니다.

고객 시연 가능 여부: 조건부 가능 — 아래 P0 이슈 3건만 해결하면 프로토타입 데모는 가능합니다.

상용 출시 준비도: 미완 — P0·P1 전체 해결 + 추가 2주 QA 후 베타 출시 권고.

🔴 P0 치명 — Toolbar 포화

현재 Toolbar에 버튼이 28개 이상 몰려 있습니다. 첫 화면을 보는 고객이 "이게 뭐지?"라는 인상을 받습니다. ETAP 처음 켤 때와 동일한 충격이지만, ETAP은 브랜드 신뢰가 있어 감내됩니다.

🔴 P0 치명 — 빈 캔버스 온보딩 없음

새 프로젝트를 만들면 아무것도 없는 흰 화면만 표시됩니다. 신규 고객은 "어떻게 시작하지?"를 혼자 해결해야 합니다. Welcome 화면 또는 퀵스타트 가이드가 필수입니다.

🔴 P0 치명 — Results Panel 탭 14개

Results Panel에 탭이 14개입니다. 화면에 다 표시되지 않아 가로 스크롤이 필요합니다. 고객은 "결과가 어디 있지?"를 찾아 헤맵니다.

"엔지니어링 툴은 복잡해도 된다는 편견이 있지만, 실제로는 ETAP·AutoCAD 같은 제품도 수십 년간 UI를 꾸준히 다듬었다. 첫 시연에서 고객이 툴을 신뢰하게 만드는 것은 기능이 아니라 인터페이스의 자신감이다."

— UX Design Lead, PowerFlow Analyzer 검토단
02

시각 언어 & 색상 시스템

일관성 평가 및 개선 방향

✅ 잘 된 점

index.css에 CSS Custom Properties 기반 색상 토큰이 정의되어 있습니다 (--navy, --ok, --warn 등). 기반 설계는 올바릅니다.

🔴 치명 — 토큰 사용률 5% 미만

CSS 변수가 정의되어 있지만 대부분의 컴포넌트가 inline style에 하드코딩된 hex값을 사용합니다. '#1e3a7a', '#152d60', '#f0f2f5' 등이 파일 전체에 수십 회 반복됩니다. 브랜드 색상 변경 시 수백 개를 수동으로 고쳐야 합니다.

⚠ 경고 — 색상 값 50+ 가지 혼용

코드베이스 전체에서 사용되는 고유 색상값은 약 50가지 이상입니다. #1a2030, #1a2838, #0a1a2a 같이 거의 구분 불가능한 어두운 텍스트 색상이 모두 다르게 정의되어 있습니다.

현재 vs 권장 팔레트

현재 사용 중인 핵심 색상 (정리 필요)

#0f2a60Navy Dark
#1e3a7aNavy Base
#3b82f6Accent
#16a34aOK Green
#ca8a04Warning Amber
#dc2626Error Red

권고: 위 6가지 + 캔버스 배경(#ffffff) + 패널 배경(#f0f2f5) + 테두리(#c8d0d8) 의 9토큰 시스템으로 정리하고, 모든 inline color를 CSS 변수로 교체. 작업공수: 1인 3일.

ℹ 전압 레벨 색상 코딩 — 잘 됨

154kV=어두운 빨강, 22.9kV=어두운 파랑, 0.4kV=어두운 초록 — IEC/ETAP 관례에 맞는 직관적인 색상 코딩입니다. 유지할 것.

ℹ 상태 색상 코딩 — 잘 됨

Load Flow 결과 오버레이 (녹=정상, 황=주의, 적=위험), Cable 부하율 바 색상이 일관됩니다. 유지할 것.

03

Toolbar / Ribbon 디자인 검토

가장 시급한 개선 영역 — 첫인상 결정 요소

🔴 P0 치명 — 버튼 28개 포화 상태

현재 Toolbar 그룹 구성:

프로젝트(5) · 파일(2) · 편집(3) · 레이아웃(1) · 해석계산(8) · 출력(2) · 도구(7) = 총 28버튼 + 상태 영역

1920px 화면에서도 그룹 레이블이 잘리기 시작하며, 1366px (노트북) 에서는 overflow가 발생합니다. 고객 데모 환경은 주로 노트북입니다.

현재 Toolbar (개략적 표현): ┌─────────────────────────────────────────────────────────────────────────────────────────────────────┐ │ [PowerFlow] │ New 저장 열기 다른이름 최근 │ 예제 초기화 │ 실행취소 다시 AutoLF │ AutoLayout │ │ │ ANALYZER │ ──── 프로젝트 ──── │── 파일 ── │──── 편집 ─────────── │─레이아웃─ │ │ │ │ LF(L) LF(A) SC N-1 조화 케이블 비대칭 │ PDF PNG │ 데이터시트 탭 부하목 SLD 도면라 모터 │ │ │ ──────────── 해석계산 ────────────── │─ 출력 ─ │──────────────── 도구 ──────────────── │ └─────────────────────────────────────────────────────────────────────────────────────────────────────┘ ↑ 1366px 화면에서 "도구" 그룹 버튼들이 잘림
❌ 현재 문제점
  • 그룹 레이블 간격에 비정상 공백 ("해 석 계 산") — 비전문적
  • 해석 버튼 8개가 동일 그룹 — 우선순위 구분 없음
  • Auto LF 토글이 "편집" 그룹 — 맥락 부적절
  • 탭 최적화·역률보상·SLD가져오기·도면라이브러리·Motor List·부하목록표 6개가 "도구"로 뭉뚱그려짐
  • 최소 높이 64px — 버튼 텍스트가 작아 읽기 불편
  • Logo 영역 160px 고정 — 좁은 화면에서 낭비
✅ 권장 구조
  • 그룹을 4개로 줄임: 파일·편집·해석·가져오기
  • 해석은 빈도 높은 순으로 LF·SC 먼저, 나머지는 ▼ 드롭다운
  • 도구 기능은 "가져오기" 그룹 또는 메뉴로 이동
  • Auto LF 토글은 ResultsPanel 헤더 우측으로 이동
  • Toolbar 높이 72px로 증가, 버튼 min-width 54px
  • 그룹 레이블 정상 텍스트 ("파일", "편집", "해석", "가져오기")
권장 Toolbar (재구성안): ┌───────────────────────────────────────────────────────────────────────────────────┐ │ [⚡ PowerFlow] │ New 열기 저장 최근 │ ↩ ↪ │ LF(Local) SC LF(API) SC ▾ │ 가져오기 ▾ │ │ ANALYZER │ ──── 파일 ──── │편집 │ ───── 해석계산 ─────────── │ Import │ │ [Auto ● 켜짐] │ └───────────────────────────────────────────────────────────────────────────────────┘ ↑ 1366px에서도 모든 버튼 표시 가능. 주요 기능 즉시 접근.
⚠ 그룹 레이블 비전문성

현재 "해 석 계 산", "도 구" 처럼 글자 사이에 공백이 들어있습니다. Microsoft Office나 ETAP도 이런 스타일을 쓰지 않습니다. "해석", "도구"로 변경하거나 영어 "Analysis", "Tools"로 통일하세요.

04

장비 팔레트 & SLD 캔버스

핵심 작업 영역 — 가장 많은 시간을 보내는 곳

✅ 팔레트 — IEC 심볼 적절

Bus(사각 바), Transformer(접원), Breaker(X 박스), Motor(M 원), Generator(G+물결), Load(삼각형)이 모두 IEC 60617 표준을 잘 따릅니다. 엔지니어들이 즉시 인식합니다.

⚠ 팔레트 — 접기/펼치기 없음

현재 7개 장비만 있지만, 향후 확장 시(콘덴서, 리액터, CT/PT 등) 스크롤이 필요해집니다. Accordion 섹션 도입 권고: Power Devices / Protection / Measurement.

⚠ 팔레트 — 검색 기능 없음

장비 30개 이상 추가 시 검색 필드가 필수입니다. ETAP/AutoCAD 모두 검색 지원. 지금 추가하지 않으면 나중에 구조를 바꿔야 합니다.

✅ 캔버스 — 순백 배경 탁월

흰 배경 + 도트 그리드는 CAD 드로잉 페이퍼를 연상시킵니다. 엔지니어에게 친숙하고 전문적입니다. 유지하세요.

⚠ 캔버스 — 빈 화면 온보딩 없음

첫 실행 시 3가지 시작 옵션 표시 권고: ① 예제 로드, ② 새 프로젝트 시작, ③ 파일 열기. ETAP/Visio 스타일의 Welcome 오버레이로 구현.

ℹ 캔버스 — 연결 핸들 발견하기 어려움

현재 핸들이 transparent입니다. 마우스 호버 시에만 연결 가능한 포인트를 알 수 있습니다. 호버 시 파란색 점 표시로 변경하면 디스커버리가 크게 향상됩니다.

05

Properties Panel

장비 속성 편집 — 반복적이고 중요한 작업

✅ Section 기반 폼 구조 — 좋음

"Identification", "Rating", "Impedance", "Tap Changer" 등 섹션 구분이 엔지니어의 멘탈모델과 일치합니다. IEC 데이터시트 구조와 동일하여 친숙합니다.

✅ Library 버튼 — 실용적

"Load From Library" 버튼이 폼 상단에 배치되어 접근성이 좋습니다. LS Cable/현대일렉트릭 데이터 불러오기 흐름이 직관적입니다.

⚠ 패널 너비 260px — 너무 좁음

Breaker RelaySettings 섹션처럼 필드가 많을 때 label(110px) + input이 좁아 입력이 불편합니다. 최소 280px, 사용자 리사이즈 지원 권고.

⚠ 값 변경 후 피드백 없음

필드를 수정해도 "저장됨"/"변경됨" 표시가 없습니다. 엔지니어는 자신의 입력이 적용되었는지 불확실합니다. 입력 후 필드 테두리가 2초 파란색→초록색으로 전환 권고.

⚠ Tab 키 네비게이션 불완전

Section이 <div>로만 구분되어 Tab으로 다음 섹션 이동 시 focus가 예측 불가능합니다. 빠른 데이터 입력이 핵심인 엔지니어에게 치명적.

ℹ "삭제" 버튼 위치 위험

패널 우상단의 "삭제" 버튼이 선택된 장비 유형 배지 바로 옆에 위치합니다. 잘못 클릭하기 쉽습니다. 확인 다이얼로그 없이 즉시 삭제는 UX 안티패턴 (Undo가 추가되어 완화되었으나 여전히 부적절한 위치).

06

Results Panel & 데이터 테이블

해석 결과 시각화 — 제품의 핵심 가치를 전달하는 영역

🔴 P0 치명 — 탭 14개, 가로 스크롤 필요

현재 탭: Load Flow · Short Circuit · Asym.Fault · Protection · Motor Starting · Coordination · Arc Flash · N-1 Cont. · TCC Viewer · Harmonics · Cable Sizing · V Profile · 역률 보상 · Study Case

1920px 화면에서도 모든 탭이 표시되지 않습니다. 탭을 발견하기 위해 스크롤이 필요한 것은 심각한 UX 문제입니다.

권장: 탭 그룹화 (2-Level Tab)

Fault · Arc Flash 보호협조 서브탭: Protection · Coordination · TCC Viewer 분석 서브탭: N-1 Cont. · Harmonics · Cable Sizing
⚠ 패널 기본 높이 248px 부족

데이터 테이블에서 보이는 행이 4~5개에 불과합니다. 드래그 리사이즈가 추가되었지만 기본값을 320px로 증가하고 최대 화면의 60%까지 허용 권고.

ℹ 테이블 행 클릭 → Canvas 선택 연동 — 좋음

결과 테이블의 행을 클릭하면 Canvas의 해당 노드가 선택됩니다. 이 interaction은 엔지니어링 툴에서 기대하는 핵심 UX로, 잘 구현되어 있습니다. 유지 + 홍보할 것.

ℹ 테이블 font-size 9.5px — 읽기 불편

데이터 테이블 셀 폰트가 9.5px입니다. 100% 줌 기준으로 노안 있는 엔지니어(40대+)가 읽기 불편합니다. 최소 11px, DataTable 컴포넌트에 사용자 설정 폰트 크기 옵션 추가 권고.

07

다이얼로그 & 모달 시스템

10개 이상의 모달이 혼재 — 일관성 검토

✅ 다크 블루 헤더 그라디언트 — 일관됨

모든 모달이 동일한 linear-gradient(to bottom, #1e3a7a, #152d60) 헤더를 사용합니다. 브랜드 아이덴티티가 일관적으로 유지됩니다.

✅ 배경 오버레이 — 일관됨

모든 모달의 배경 오버레이가 rgba(0,0,0,0.42~0.44)로 통일되어 있습니다.

⚠ z-index 체계 혼란

모달별 z-index: 9200, 9300, 9350, 9400, 9500, 9600 — 6단계가 임의로 분산되어 있습니다. Toast가 9999, TapOptimizer가 9400 등 체계가 없습니다. z-index 맵 문서화 필요.

⚠ border-radius 불일치

다이얼로그에 borderRadius: 2, 3, 4 세 가지가 혼용됩니다. 8px (카드)·4px (입력)·2px (배지) 3단계로 통일 권고.

세부 디테일이 전문성 인식을 결정한다

🔴 P0 — 폰트 크기 스케일 22단계

코드베이스에서 사용 중인 폰트 크기: 7px, 7.5px, 8px, 8.5px, 9px, 9.5px, 10px, 10.5px, 11px, 12px, 13px, 14px, 15px, 16px, 22px, 28px, 32px, 40px... 사실상 type scale이 없습니다. 화면을 혼잡하게 보이게 만드는 핵심 원인입니다.

권장 Type Scale (7단계)

xs: 10px / 1.4 — 배지, 메타 정보, 상태바 sm: 11px / 1.5 — 테이블 셀, 폼 레이블, 팔레트 설명 base:12px / 1.6 — 기본 UI 텍스트, 버튼 레이블 md: 13px / 1.6 — 섹션 헤더, 강조 텍스트 lg: 16px / 1.4 — 패널 타이틀, 모달 헤더 xl: 20px / 1.3 — 페이지/다이얼로그 제목 2xl: 28px / 1.2 — 랜딩/Welcome 화면 헤드라인 숫자/계측값: Consolas 11px/12px만 사용
✅ 아이콘 — 커스텀 SVG 일관성 좋음

모든 아이콘이 인라인 SVG로 통일되어 있습니다. 외부 아이콘 라이브러리 의존 없음. IEC 심볼과 UI 아이콘의 스트로크 두께(1.2~1.5px)가 잘 통일되어 있습니다.

⚠ 툴팁 거의 없음

대부분의 버튼에 title 속성이 없어 아이콘 의미를 마우스 오버로 확인할 수 없습니다. 특히 Toolbar 아이콘들이 모호합니다. 모든 icon-only 버튼에 title 속성 추가 필수.

✅ Undo/Redo (Ctrl+Z/Y) — 추가됨

50스텝 히스토리, 키보드 단축키, Toolbar 버튼. 실수를 두려워하지 않고 작업할 수 있습니다.

✅ 인라인 이름 편집 — 추가됨

노드 더블클릭으로 이름 편집. 엔지니어의 반복 작업을 크게 단축합니다.

✅ Toast 알림 시스템 — 추가됨

연결 실패, 삭제 확인, 탭 최적화 적용 결과가 비침투적으로 표시됩니다.

🔴 분석 실행 중 진행 상태 없음

Load Flow 실행 중 Toolbar에 "◌ 계산 중…" 텍스트만 표시됩니다. 500개 모터 네트워크에서 3초 이상 걸릴 때 "앱이 멈췄나?" 오해를 유발합니다. 프로그레스 바 또는 스피너가 캔버스 오버레이에 필요합니다.

⚠ 키보드 단축키 없음 (해석)

F5=Load Flow, F6=Short Circuit 같은 단축키가 없습니다. ETAP은 Ctrl+F5, Ctrl+F6 등을 지원합니다. 반복 작업에서 마우스 이동 거리가 큰 낭비입니다.

⚠ 접근성 (WCAG AA) 미검증

배경 #f0f2f5 + 텍스트 #8a9aaa는 대비비 2.3:1로 WCAG AA 기준(4.5:1) 미달입니다. 레이블, 배지, 상태 텍스트 다수가 대비 기준을 위반합니다.

진행 상태 오버레이 — 계산 중 "◌ 계산 중…" 텍스트를 반투명 오버레이로 교체. 엔지니어: "500개 모터 LF 돌릴 때 2~3초 걸린다" → 오버레이 필요성 동의.
